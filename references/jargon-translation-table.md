# Jargon Translation Table

## Backend and Infrastructure

| Jargon | Plain-English Replacement |
|--------|---------------------------|
| middleware | code that runs between the user request and the server response |
| endpoint | a specific web address that accepts requests |
| database migration | a planned change to the structure of stored data |
| pre-save hook | automatic instructions that run before data is saved |
| aggregation pipeline | a multi-step database query that groups and transforms records |
| index | a fast lookup structure for database searches |
| shard | a separate section of a database that holds part of the total data |
| replica set | a group of database copies that stay in sync for safety |
| load balancer | a traffic director that spreads requests across multiple servers |

## Frontend and User Interface

| Jargon | Plain-English Replacement |
|--------|---------------------------|
| hydration | the process of making a server-built page interactive in the browser |
| component | a reusable building block of a web page |
| prop | a piece of data passed into a component |
| state | the current data held by a component while the user is interacting |
| re-render | rebuilding part of the page when data changes |
| layout | the overall structure and arrangement of a page |
| route | a unique page address within the application |

## Authentication and Security

| Jargon | Plain-English Replacement |
|--------|---------------------------|
| token | a digital pass that proves who a user is |
| session | a temporary record of a user's login on the server |
| hash | a scrambled version of data that cannot be reversed |
| salt | extra random data added before scrambling to increase security |
| OAuth | a standard way to log in using another trusted service |
| scope | the specific permissions granted to an app |
| realm | a defined security boundary or area of control |

## Development Workflow

| Jargon | Plain-English Replacement |
|--------|---------------------------|
| refactor | restructure existing code without changing what it does |
| deprecation | marking a feature as outdated and planned for removal |
| lint | automatically check code for style and error rules |
| type check | verify that data shapes match their expected definitions |
| unit test | a small automatic check that one piece of code works |
| end-to-end test | an automatic check that simulates a full user journey |
| merge conflict | a situation where two people changed the same part of a file |
| cherry-pick | copy one specific change from one branch to another |
| rebase | replay changes on top of a newer version of the code |

## General Computing

| Jargon | Plain-English Replacement |
|--------|---------------------------|
| cache | temporary fast storage for recently used data |
| queue | a waiting line for tasks to be processed in order |
| worker | a background process that handles tasks outside the main flow |
| webhook | an automatic message sent from one system to another when something happens |
| payload | the actual data being sent inside a message |
| schema | the defined shape and rules for how data must be structured |
| serialization | converting data into a format that can be stored or sent |
| deserialization | converting stored or sent data back into a usable form |
